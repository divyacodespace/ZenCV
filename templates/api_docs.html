Backend API

Programmatic JSON access to the resume analyzer and JD matcher.

POST /api/analyze

Upload a PDF/DOCX resume; receive the full analysis as JSON.

Request (multipart/form-data)
  • resume — file (.pdf or .docx, < {{ 8 }} MB) required
Example
curl -X POST http://127.0.0.1:5000/api/analyze \
  -F "resume=@resume.pdf"
Response (200)
{
  "filename": "resume.pdf",
  "word_count": 612,
  "ats_score": 78,
  "sections_found": ["contact", "summary", ...],
  "sections_missing": ["projects"],
  "skills_found": { "Programming": ["python", "java"], ... },
  "skills_missing": { "Cloud & DevOps": ["aws", "docker", ...] },
  "keyword_gaps": ["aws", "docker", ...],
  "formatting_issues": [],
  "strengths": [...],
  "weaknesses": [...],
  "suggestions": [...],
  "breakdown": { "Sections": {"score": 22, "max": 25, "detail": "..."} , ... }
}
POST /api/jd-match

Compare a resume against a job description.

Request (multipart/form-data)
  • jd_text — job description text required
  • resume — file (.pdf or .docx) or
  • resume_text — raw text of the resume
Example
curl -X POST http://127.0.0.1:5000/api/jd-match \
  -F "resume=@resume.pdf" \
  -F "jd_text=Senior Python engineer with AWS and Docker..."
Response (200)
{
  "match_score": 72,
  "matched_keywords": ["python", "flask", ...],
  "missing_keywords": ["kubernetes", "terraform", ...],
  "matched_skills": ["python", "flask"],
  "missing_skills": ["kubernetes", "terraform"],
  "jd_word_count": 412,
  "resume_word_count": 612,
  "top_jd_terms": [["python", 8], ["aws", 6], ...],
  "verdict": "Decent match — add a few missing keywords to push higher.",
  "suggestions": [...]
}
Errors

All errors return JSON with HTTP 400:

{ "error": "missing or invalid 'resume' file (.pdf or .docx)" }
